# TwinSeam Deals - Sporting Goods Deal Aggregator

## Overview
TwinSeam Deals is a sporting goods deal aggregator designed to help users find deeply discounted products (around 50% off MSRP) from various retailers, with a prominent feature for Twin Seam Sports' own deals. The platform enables users to personalize their deal-finding experience by selecting preferred sports, equipment types, and condition preferences, receiving multiple daily deal notifications. The project's vision is to become the go-to resource for sports enthusiasts seeking quality gear at competitive prices, while also expanding Twin Seam Sports' market presence.

## System Architecture
The application utilizes a React, Vite, and TailwindCSS frontend, served by an Express.js backend. PostgreSQL, managed by Drizzle ORM, serves as the database. Authentication is handled via Replit Auth (OIDC). The UI/UX emphasizes a clean, modern design built with Shadcn/UI components.

### Technical Implementations
- **Frontend**: React + Vite + TailwindCSS + Shadcn/UI for a responsive and modern user interface.
- **Backend**: Express.js provides the API layer.
- **Database**: PostgreSQL (Neon-backed) with Drizzle ORM for schema definition and Zod for validation. GIN trigram indexes on `deals.title` and `deals.brand` for fast ILIKE search. Composite indexes on `(sport_id, equipment_type_id)` and `(source_id, sport_id)`. Index on `last_seen_at` for staleness queries. Automated stale deal cleanup runs on startup (batched, max 200K/run) and daily at 3am ET: marketplace deals (eBay, SidelineSwap) >7 days, retailer deals >14 days.
- **Authentication**: Dual system — Magic Link (primary, works on custom domains) + Replit Auth OIDC (fallback). Sign in by **email** or **phone (SMS)** via 6-digit codes (15-min expiry): `POST /api/auth/magic-link/send|verify`, `POST /api/auth/sms/send|verify`. Tables `magic_links`, `sms_auth_codes`; `users.phone` (unique, nullable). Rate-limited (5 sends / 10 verifies per 5-min per identifier+IP); code returned to client only in dev. Email via **SendGrid**, SMS via Twilio Messaging Service. Session: `{magicLink, userId, email, phone}`. Modules `server/magic-link-auth.ts`, `server/sms-auth.ts`; component `MagicLinkDialog.tsx` (Email/Phone switcher with a password toggle). **Optional password login** (scrypt, `users.password_hash`/`password_salt`): `POST /api/auth/password/login|set`, `GET /api/auth/password/status` (`timingSafeEqual`, min 8 chars, rate-limited). **Security**: `toPublicUser()` (`shared/models/auth.ts`) strips `passwordHash`/`passwordSalt` from EVERY user serialization — never spread a raw user into a response.
- **Deal Filtering & Curation**: Deals are filtered by sport, equipment, condition, and discount. A manufacturer MSRP verification system ensures accurate discount display. A curated default feed prioritizes deals from core sports with significant discounts. Admin-curated featured deals and automatically included high-demand products enhance deal visibility.
- **Dynamic Content**: Equipment sub-filters allow granular deal categorization. Brand filters dynamically update based on sport selection.
- **Notifications**: Users can receive push and SMS notifications for deal alerts. Public SMS opt-in page at `/notifications` with TCPA-compliant consent flow (unchecked checkbox, disclosure text, confirmation SMS). SMS templates use "TSSDeals" branding with STOP/HELP language. Rate-limited public subscribe endpoint at `POST /api/sms/subscribe`. Public subscribers are persisted in the `sms_subscribers` table (phone unique, marketing/transactional consent flags, status, opt-in IP/time); the STOP webhook marks both app users (`optOutSmsByPhone`) and subscribers (`optOutSmsSubscriberByPhone`) unsubscribed.
- **SMS Deal Blast** (admin): Paste a retailer link → auto-fetch OpenGraph images + AI write-up (GPT-4o) → campaign with a short slug + public landing page `/d/:slug` → "Push SMS" to marketing-consented recipients (`sms_subscribers` marketing+active UNION app users with `smsEnabled`, deduped by last-10 digits via `listMarketingRecipients`). Sent text = admin copy + landing short link + "Reply STOP to opt out". Table `sms_campaigns`. Module `server/sms-campaigns.ts` (SSRF-hardened `fetchLinkPreview`: DNS-resolve + block private/loopback/link-local/CGNAT/metadata IPs, redirect re-validation, 2MB/text-html cap; plus `generateWriteup`/`generateSlug`). Routes: `POST /api/admin/sms-campaigns/fetch-preview`, `GET/POST/DELETE /api/admin/sms-campaigns`, `POST /api/admin/sms-campaigns/:id/send`, public `GET /api/campaign/:slug`. UI in `Admin.tsx`; public page `DealBlastPage.tsx`.
- **Deal Insights**: Each deal includes a price history chart and displays "Historical Low" badges. Users can set "Price Alerts" for specific deals or products across sellers.
- **Deal Hiding**: Authenticated users can hide individual deals (EyeOff button on each card). Hidden deals are filtered server-side on all subsequent `GET /api/deals` calls. Table: `hidden_deals` (user_id + deal_id composite PK). Routes: `POST /api/deals/:id/hide`, `DELETE /api/deals/:id/hide`. The deal card immediately disappears on hide; deals can be unhidden by calling the DELETE route.
- **Search**: Multi-word search uses OR logic (any matching term surfaces a result), with relevance sorting (exact phrase match in title ranks first, then by discount/date). Powered by ILIKE with GIN trigram indexes in production.
- **Popular Products**: Admin-curated + auto-trending products shown on landing page, deals feed, and `/deals` hub. Table: `popular_products` (unique slug index). Admin CRUD at `GET/POST/PATCH/DELETE /api/admin/popular-products`. Public API: `GET /api/popular-products` merges admin picks (sorted by `sort_order`) + trending from `deal_clicks` (30-day window), max 12 total. Seeded with 12 initial products across 6 sports.
- **Admin Tools**:
    - **Bonus Deals**: Admin-managed section for non-sporting goods deals.
    - **eBay Listing Assistant**: AI-powered tool (OpenAI GPT-4o Vision) to generate optimized eBay listings from photos and descriptions, integrating with the eBay Inventory API.
    - **eBay Pricing Analysis**: Admin tool for comparing Twin Seam Sports' eBay listings with market data, generating daily pricing recommendations, and tracking profit margins.
    - **Promo Codes & Coupons**: Auto-syncs promo codes from CJ, Impact, Fanatics, Rakuten. Codes are matched to deals by advertiser. Users see a "Code: XXXX" badge on deals (click-to-copy). Admin CRUD, manual entry, sync trigger. Table: `promo_codes`. Deal fields: `promoCode`, `promoDescription`. Module: server/promo-codes.ts. API: GET/POST/PATCH/DELETE `/api/admin/promo-codes`, POST `/api/admin/promo-codes/sync`, GET `/api/admin/promo-codes/stats`.
    - **Analytics Dashboard**: Provides administrators with insights into click tracking, user visits, top deals, and user demographics. Emails `justin@twinseamsports.com` and `jshirk1@gmail.com` are excluded from all analytics (heartbeat tracking, click stats, visit stats, and user list).
    - **AI MSRP Verification**: Utilizes OpenAI GPT-4o-mini to verify manufacturer MSRP from deal titles, caching results and updating deal displays with verified discounts.
- **SEO Deal Pages**: Auto-generated pages at `/deals/:slug` for sports, categories, and brands. Category hub at `/deals` with browsable directory. Price comparison tables, SMS signup CTAs, breadcrumb navigation, and internal linking. Backend: `GET /api/seo/slugs`, `GET /api/seo/page/:slug` with 5-min cached slug resolution (sport → category → brand → product search). Sitemap at `/sitemap.xml` with caching and deduplication. Component: `client/src/pages/DealPage.tsx`, SEO metadata via `client/src/components/Seo.tsx`.
- **Private Team Stats Pages** (`/team/:slug`): Password-gated baseball stats tracker (hidden from nav, direct URL only). Module: `server/team-stats.ts`; component: `client/src/pages/TeamPage.tsx`. Uses `xlsx`, `recharts`, `openai`. Seeded team: Knox Stars 7U (slug `stars7u`; legacy `knox-stars-7u` auto-migrated on startup; bootstrap password via env `BB_KNOX_PASSWORD`).
    - **Schema**: `bb_teams`, `bb_players`, `bb_games`, `bb_player_game` (one row per player+game+source, unique `(game_id, player_id, source)`, source = `manual` | `gamechanger`; holds hitting/fielding/pitching, plus `batting_order` and `starting_position`). Per-position fielding detail in `bb_player_fielding` (unique `(game_id, player_id, position, source)`) and player-less `bb_team_fielding` (unique `(game_id, position, source)`). Team admins in `bb_team_admins` (`(team_id, lower(email))`).
    - **Auth / access**: scrypt-hashed team password → `req.session.bbTeamAccess[teamId]` (30-day TTL). TSS admin (`justin@twinseamsports.com`) and any granted team admin bypass the gate and get the Admin tab; `requireTeamAdmin`/`requireTeamAccess` both honor grants. Manage admins via `GET/POST/DELETE /api/team/:slug/admin/admins`.
    - **Viewer**: mode toggle `?mode=manual|combined|gamechanger` (`combined` prefers manual per `(gameId, playerId)`, falls back to GameChanger for gap-filling). Tabs (Leaderboard, Games, Pitching, Fielding, Compare, Trends) are scoped by a Season selector (`?season=`). Sortable headers via shared `useSort`/`SortHead`/`sortRows` (nulls last). **Compare** (`GET /api/team/:slug/compare?gameIds=`) shows per-player Manual / GC / Δ for games that have BOTH sources. **Trends** (`GET /api/team/:slug/trends?season=&mode=&window=`) recharts per-player offensive + defensive trajectories (no pitching) with Per-game / Cumulative(window) / Cumulative(season) views.
    - **Stat math**: AVG, OBP (denom AB+BB+SAC), SLG, OPS, ISO, BABIP, BB%, K%, XBH/XBH%, Secondary Average; FPCT; ERA + K/9 (normalized to 9 innings), WHIP; IP stored as `pitching_outs` int (5.2 IP = 17 outs). **AB convention**: this team enters **AB as raw total plate appearances** (walks/sacrifices/HBP/SF NOT subtracted), so AVG/OBP/SLG/OPS read lower than official figures — disclaimers shown on Leaderboard, per-game dialog, and all entry/upload/scan surfaces.
    - **Fielding by position**: `bb_player_game.po/a/e` is the authoritative fielding TOTAL (powers Leaderboard + Fielding tab); `bb_player_fielding` holds per-position DETAIL. `GET /api/team/:slug/fielding-by-position?season=&mode=` aggregates by position and FALLS BACK to the cached total when a `(game, player, source)` has no detail rows (no backfill needed); `bb_team_fielding` rows surface as a synthetic `__team__` "Team (unattributed)" player. Admin editors replace-set splits and recompute the cached total + primary position. **Any change to a po/a/e total must delete/re-parent the matching split rows** — enforced on the stats PUT, Excel/GameChanger re-import, stat-row delete, and game merge; FK cascade on game delete.
    - **Admin data entry**: roster & game CRUD; Manual Stat Entry, Bulk Stat Entry (`PUT /api/team/:slug/admin/stats/bulk`), and the Game Details inline editor (all expose batting Order + Start position; column-level omit-empty partial upsert so a blank field never nulls an existing value or wipes fielding splits — the standalone card does NOT prefill, the inline editor does). Excel template (`/api/team/:slug/admin/template.xlsx`) + upload (dedupes games by `(date, lower(opponent))`, tags rows by source). **Scorebook Scan** (`POST /api/team/:slug/admin/scan-scorebook`, GPT-4o Vision) parses the totals column, auto-matches players by jersey (name fallback), returns editable rows committed via the manual upsert.
    - **Seasons**: every `bb_games` row carries a `season` tag; the team's `season` is the **active** season, stamped onto all new games (manual, Excel, GC import, scan). Switch via the "Active Season" admin card (`PATCH /api/team/:slug/admin/season`); list via `GET /api/team/:slug/seasons`. Pre-existing data backfilled to `"Spring 2026"` on bootstrap.
    - **AI tools** (GPT-4o, `enforceSameOrigin` + `OPENAI_API_KEY`, print-to-PDF via `window.print()`): **Lineup Generator** (admin-only, `POST /api/team/:slug/admin/ai-lineup`) returns both a Coach-view and All-view lineup from offensive aggregates + averaged poll speed/brIQ. **Player Evaluations** (`window season|last10|last5`) and **Season Report** (body `{ scheme: 'auto'|'thirds'|'quarters', coachPitch? }`, adaptive chronological periods) are **dual-audience**: each is a factory handler registered twice — a **family** viewer route (`POST /api/team/:slug/ai-evaluate|ai-season-report`, `requireTeamAccess`, warm/encouraging tone, improvements optional) and a **coach** admin route (`POST /api/team/:slug/admin/ai-evaluate|ai-season-report`, `requireTeamAdmin`, candid/critical tone, always ≥1 improvement). **Rankings are coach-only**: the family UI hides all rank numbers + the offensive/defensive ranking tables and lists players alphabetically (never by performance), and the family prompt forbids ranking/standings language in the prose; coaches still get the numbered rankings. **Tone is decided server-side by which route is hit (audience bound at registration); the client cannot request a tone.** Server-computed stats are identical across audiences. Frontend: shared `AIEvaluationCard`/`SeasonReportCard` take an `audience` prop selecting endpoint + copy; admin Admin tab passes `coach`, and a viewer **Reports** tab (visible to all access) passes `family`. All numbers are server-authoritative (GPT only supplies narrative/trend/smallSample + offensive/defensive rankings, every playerId validated and hallucinated IDs filtered); defensive ranking only when fielding sample is large enough (`maxChances ≥ 10` AND ≥3 players with ≥5 chances). **Coach-pitch aware**: ≤8U (derived from the `NU` age in the team name) omits walk/steal/pitching commentary and drops bb/sb/bbRate/secAvg from the AI payload (still covers OBP/contact/XBH/RBI/K); admin can override via `coachPitch`.
- **Data Structures**: Supports a wide range of predefined and user-created sports (including Tennis, Pickleball, Badminton, Squash added for Wilson integration), equipment types, and over a hundred retailers, tracking detailed deal information.

## External Dependencies
- **Replit Auth (OIDC)**: User authentication.
- **PostgreSQL (Neon)**: Cloud database solution.
- **OpenAI API (GPT-4o Vision, GPT-4o-mini)**: Powers the eBay Listing Assistant and AI MSRP Verification.
- **Twilio**: Enables SMS notifications.
- **CJ Affiliate**: Programmatic access to retailer product data with partner-specific IDs.
- **eBay APIs (Browse, Inventory, Offer, OAuth2)**: Used for searching listings, creating/publishing eBay listings, and managing user authentication.
- **Shopify API**: Integrates with Twin Seam Sports' and Baseball Resale's Shopify stores.
- **WooCommerce Store API**: Syncs products from NameOfTheGame.com.
- **SidelineSwap API**: Integrates deals from the SidelineSwap marketplace. Also supports the SidelineSwap Inventory API (partner access) for pushing eBay listings to SidelineSwap. Requires env vars `SIDELINESWAP_API_KEY` and `SIDELINESWAP_CLIENT_ID`. Admin tool at SidelineSwap Sync section. Module: `server/sidelineswap-inventory.ts`. DB table: `sidelineswap_syncs`. Routes: `GET /api/ebay/inventory`, `GET/POST /api/admin/sidelineswap-sync`, `POST /api/admin/sidelineswap-sync/batch`, `GET /api/admin/sidelineswap-sync/categories`, `GET /api/admin/sidelineswap-sync/addresses`, `DELETE /api/admin/sidelineswap-sync/:id`.
- **ShareASale API**: Affiliate product search.
- **Impact.com API**: Catalog-based product feed. Includes Wilson Sporting Goods integration with smart per-item sport detection (tennis, pickleball, badminton, squash, volleyball, basketball, football, baseball, etc.) from Impact catalog Category/SubCategory fields. Admin manual sync at `POST /api/admin/sync/impact`. Sports Tennis, Pickleball, Badminton, and Squash were added to the DB specifically for Wilson's multi-sport catalog.
- **Rakuten Advertising API**: Product search with merchant-specific sync for brands like Hoka and Orvis.
- **Amazon PA-API 5.0**: Product search.
- **Play It Again Sports (scraper)**: National BigCommerce (Stencil) catalog of mostly used baseball & softball gear. No affiliate feed, so deals are scraped from server-rendered category listing pages (`/baseball-and-softball/<slug>/?limit=50&page=N`, ~50 cards/page, paginated). Scope = core gear only (bats, gloves, catcher's gear, helmets, bags, balls, batting gloves, training aids; apparel/cleats/socks/umpire/field equipment excluded). Per-item condition from the title prefix ("New" → new, else preowned); sport refined for fastpitch/slowpitch. RRP markers are almost always empty for resale items, so most deals carry only a sale price (msrp/percentOff null), consistent with the other no-affiliate scrapers. Source id `play-it-again-sports`. Module `server/playitagain-sync.ts` (`syncPlayItAgain`, exported `parseListingCards`); scheduler wrapper runs in the 4x/day full sync; manual trigger `POST /api/admin/sync/playitagain` + card in `Admin.tsx`. Stale cleanup covers it as a retailer source (>14 days).
